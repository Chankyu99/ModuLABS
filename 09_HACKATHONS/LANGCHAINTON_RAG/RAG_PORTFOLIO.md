# 기내뭐돼 | RAG 프로젝트 포트폴리오

## 1. 프로젝트 개요

- 한국·미국 항공 보안/통관 규정 84건을 검색 근거로 활용해, 여행객의 자연어 질문을 노선·물품 단위로 해석하고 반입 가능 여부를 출처와 함께 반환한 도메인 특화 RAG 챗봇 구축
- 공항 현장의 물품 폐기, 보안검색 재확인, 탑승 전 의사결정 지연을 줄이기 위한 검색 정확도 개선과 응답 속도 단축 중심 설계

### 핵심 지표

| 항목 | 수치 | 의미 |
|---|---:|---|
| 인덱싱 규정 문서 | 84건 | 한국 공항 보안 45건 + 미국 도착 통관 39건의 정책 단위 데이터베이스 구축 |
| 검색어 증강 | 1,246개 | 문서당 평균 14.83개 동의어·예상 검색어 추가를 통한 검색 표면적 확대 |
| OpenAI 호출 최적화 | 2회 -> 1회 | 슬롯 추출과 DB 매핑 로직 통합을 통한 병목 제거 |
| 평균 응답 시간 | 50초 -> 15초 | 약 3.3배 속도 개선 |

## 2. 배경

- 2024년 해외여행객 2,872만 명 규모와 인천공항 반입금지 물품 적발 약 500만 건 규모가 보여주는 현장 문제의 상시성
- 전체 출발 승객 100명 중 약 17.8명이 반입금지 물품을 소지한 상태로 공항에 도착하는 구조적 비효율
- 기존 범용 LLM의 한계인 환각, 출처 불명확, 최신 규정 반영 지연 문제
- 기존 항공사 챗봇의 한계인 타국 규정 회피, 대화 맥락 단절, 질의 전환 대응 부족 문제
- 2026년 1월 26일 시행된 보조배터리 안전 규정 강화와 같은 최신 정책 변화에 대한 빠른 반영 필요성

## 3. 해결 과정

### 3-1. 도구 및 아키텍처 구성

| 구성 요소 | 적용 기술 | 선택 근거 |
|---|---|---|
| 슬롯 추출·응답 생성 | `GPT-5-mini` (`reasoning_effort="low"`) | 구조화 출력 안정성과 응답 속도 균형 확보 |
| 일반 지식 Fallback | `GPT-5.2` | DB 미등재 물품에 대한 항공 일반 규정 추론 성능 확보 |
| 데이터 증강 | `GPT-5-mini` | 반복적 키워드 생성 작업의 비용 효율성 확보 |
| 임베딩 | `text-embedding-3-small` | 짧은 규정 문서와 한국어·영어 혼합 질의 처리의 비용 대비 성능 확보 |
| 벡터 저장소 | ChromaDB | 로컬 프로토타이핑과 메타데이터 필터 검색의 단순성 확보 |
| 오케스트레이션 | LangChain | Retriever, Document, Chat 모델 연결의 모듈성 확보 |
| UI | Streamlit | 빠른 데모 구현과 상태 기반 슬롯 UI 구성 용이성 확보 |
| 라우팅 기법 | 슬롯 필링 + 메타데이터 필터 검색 + Two-Track Fallback | 정확도와 지연 시간 동시 관리 목적 |

- 오프라인 파이프라인: 원문 규정 수집 -> 키워드 증강 -> 임베딩 -> Chroma 적재 구조
- 온라인 파이프라인: 슬롯 추출 및 DB 매핑 -> 메타데이터 필터 검색 -> 규정 기반 생성 또는 일반 지식 Fallback 구조
- 사용자 질의 누락 시 재질문, 노선 변경 시 슬롯 덮어쓰기, 미지원 국가 경고 노출까지 포함한 상태 기반 대화 흐름 설계

### 3-2. 데이터 수집 및 청킹

- 데이터 수집 범위: 한국 공항 보안 규정 45건, 미국 도착 통관·검역 규정 39건 수집
- 데이터 구조화 항목: `jurisdiction`, `item`, `stage`, `doc_type`, `cabin_decision`, `checked_decision`, `source_hint`, `evidence_url`, `doc_id` 메타데이터 정규화
- 스테이지 분리: 공항 보안(`airport_security`) 45건과 도착 통관(`customs_arrival`) 39건 분리를 통한 검색 의도 해석 용이성 확보
- 청킹 전략: 별도 텍스트 분할 없이 "규정 항목 1개 = 청크 1개" 정책 적용
- 청크 오버랩: `0` 적용

청킹을 세분화하지 않은 이유는 명확합니다. 본 프로젝트의 원문 데이터는 이미 "물품 단위 규정"으로 정리된 짧은 정책 레코드이며, 원문 평균 길이는 77.2자, 증강 후 평균 길이도 188.1자 수준입니다. 이 길이대에서 추가 분할을 적용하면 "기내 불가 / 위탁 조건부"처럼 하나의 판정을 구성하는 문장이 서로 다른 청크로 쪼개질 가능성이 높습니다. 따라서 의미 단위를 보존하는 단일 규정 청크 전략이 검색 정밀도 측면에서 더 합리적이었습니다.

- 데이터 증강 방식: 84개 문서 각각에 평균 14.83개, 총 1,246개의 동의어·예상 검색어 추가
- 증강 목적: "보조배터리 -> 리튬이온", "미숫가루 -> 농산물/가공식품"과 같은 일상어-규정어 간 표현 불일치 해소
- 검증 장치 1: 국가 메타데이터 필터를 통한 타 노선 규정 누수 차단
- 검증 장치 2: `doc_id` 기반 중복 관리와 검색 단계 `seen_ids` 처리로 중복 응답 방지
- 검증 장치 3: 규정 문서에 `source_hint`와 일부 `evidence_url` 보존을 통한 근거 추적성 확보
- 검증 장치 4: 시나리오 테스트 설계 내재화

코드 기준 시나리오 테스트는 최소 5개 케이스로 설계되어 있습니다. 물품만 말한 경우, 배터리 용량이 포함된 경우, 미지원 국가 노선 질의, 대화 도중 노선 일부 변경, 대화 도중 노선 전체 교체 상황을 모두 다루며, 이는 실제 사용자 입력 분포에서 자주 발생하는 오류 패턴을 선제적으로 커버하기 위한 검증 설계입니다.

### 3-3. 검색 및 생성 파이프라인 최적화

#### 정확도 향상과 지연 시간 단축의 연결 구조

- 검색 정확도 향상 목적: 사용자 표현을 규정 문서 표현으로 안정적으로 연결하는 회수율 개선 설계
- 지연 시간 단축 목적: 슬롯 추출과 DB 매핑을 분리 호출하던 병목 제거
- 최종 연결 결과: 검색 적중률 개선과 API 호출 수 절감의 동시 달성

구체적으로는 세 가지 설계가 핵심이었습니다.

1. 문서 증강을 통해 검색 표면적 자체를 넓힌 회수율 개선
2. 국가 메타데이터 필터로 검색 후보군을 줄인 정밀도 개선
3. 슬롯 추출과 DB 매핑을 하나의 프롬프트로 통합한 호출 비용 절감

이 구조 덕분에 검색은 더 정확해지고, 생성 단계까지 도달하는 시간은 더 짧아졌습니다. 실제로 평균 응답 시간은 50초에서 15초로 줄어들었습니다.

#### 적용 알고리즘과 프롬프트 엔지니어링

- 검색 알고리즘: Chroma `similarity_search_with_score`
- 후보 문서 수: `TOP_K = 5`
- 1차 질의 구성: `mapped_items + original item` 결합 질의
- 2차 제어 로직: 매핑 실패 시 원문 질의만 사용하되 `score <= 1.2` 임계값 적용
- 후보 축소 방식: `jurisdiction` 메타데이터 필터를 이용한 국가별 검색
- 재정렬 방식: DB 매핑 성공 시 검색된 문서 중 `item`이 실제 매핑 결과와 일치한 문서만 통과

프롬프트 설계는 단순 응답 품질보다 라우팅 안정성에 초점을 맞췄습니다.

- JSON-only 출력 강제
- 국가 미언급 시 `null` 강제
- 한국어 사용 여부만으로 출발국을 추정하지 않도록 금지 규칙 삽입
- 대화 중 새 노선 언급 시 기존 슬롯을 덮어쓰는 Dynamic Override 적용
- 최종 응답 형식 표준화
  - 첫 줄 판정 이모지
  - 세부 규정 Bullet
  - 마지막 줄 출처 링크

#### 신뢰성 검증과 환각 방지 설계

본 프로젝트 자료에는 정답셋 기반 환각률, Faithfulness, Answer Accuracy와 같은 오프라인 평가 수치가 남아 있지 않습니다. 따라서 본 포트폴리오는 "환각률 n% 감소"와 같은 과장 대신, 잘못된 확정형 답변을 방지하는 제어 장치 중심으로 신뢰성을 설명합니다.

- 규정 검색 성공 시에만 출처 기반 답변 생성
- 출발지·도착지 누락 시 재질문 후 검색 수행
- 출발지와 도착지가 동일할 경우 예외 처리
- 미지원 국가 노선일 경우 상단 경고문 선출력
- DB 매핑 자체가 실패한 경우 `GPT-5.2` 일반 지식 트랙으로 분리
- DB 매핑은 되었으나 문서 점수가 낮은 경우 "찾지 못했다"는 명시적 Fallback 메시지 출력

이 설계의 의미는 "모를 때 아는 척하지 않는 시스템" 구축입니다. 특히 국가 초깃값 `null` 강제와 미지원 국가 경고는, 언어와 정황만으로 국가를 상상해 잘못된 규정을 제시하는 오류를 구조적으로 차단한 장치였습니다.

### 3-4. 시스템 시각화 전략

평가자 친화적 아키텍처 시각화의 핵심은 "오프라인 준비 과정"과 "온라인 응답 과정"의 분리입니다.

- 상단 배치: 사용자 질의 -> 슬롯 확인 -> 라우팅 -> 응답 생성의 실시간 흐름
- 하단 배치: 원문 데이터 -> 증강 -> 임베딩 -> 벡터 저장의 오프라인 준비 흐름
- 의사결정 노드 강조: 슬롯 완결 여부, 미지원 국가 여부, 문서 발견 여부, Fallback 진입 여부
- 색상 규칙 고정: 일반 데이터 흐름, 벡터 검색, 결정 포인트를 각각 다른 색으로 구분
- 우측 종단 배치: Streamlit UI를 마지막에 두어 사용자 체감 결과와 파이프라인 연결성 강조

포트폴리오 자료 배치 전략은 다음 순서가 가장 설득력이 높습니다.

1. 문제 규모와 현장 손실 수치 제시
2. 기존 서비스 한계 비교
3. RAG 파이프라인 전체 구조 제시
4. 데이터 증강과 속도 최적화 사례 제시
5. 실제 데모 화면과 답변 예시 제시
6. 기대 효과와 향후 확장 전략 제시

## 4. 실제 결과 및 예상 임팩트

### 실제 결과

- Streamlit 기반 실사용형 데모 구현 완료
- 슬롯 상태 표시, 대화 초기화, 누락 정보 재질문, 출처 링크 포함 응답 UX 구현
- 평균 응답 시간 50초 -> 15초 단축
- 슬롯 추출과 DB 매핑 로직의 One-shot 통합 완료
- 한국·미국 규정 84건 기반 규정 검색형 챗봇 구현
- 문서당 평균 14.83개 검색어 증강을 통한 규정어-일상어 간 표현 간극 축소
- DB 매핑 성공 시 규정 기반 답변, 실패 시 일반 지식 Fallback으로 분기하는 Two-Track 구조 구현

### 비즈니스 임팩트

- 연간 약 500만 건 규모의 반입금지 물품 적발 문제에 대한 사전 예방형 정보 서비스 포지셔닝
- 공항 보안검색대 재검사와 현장 폐기 확률을 줄이는 여행 전 의사결정 지원 도구 가치
- 액체류 적발 422만 건 규모를 근거로 액체·식품·장류 카테고리 우선 대응 전략의 타당성 확보
- 김치·장류 폐기와 같은 고빈도 생활형 손실 사례를 줄이는 생활밀착형 서비스 가치

예상 임팩트는 다음과 같이 방어할 수 있습니다.

- 보수적 가정 기준, 연 500만 건 적발 중 1%만 사전 예방해도 약 5만 건의 현장 폐기·재검색 감소 여지
- 전체 출발 승객 중 17.8%가 반입금지 물품 문제를 경험한다는 수치를 고려할 때, 정보 탐색 비용을 줄이는 것만으로도 체감 UX 개선 가능성 큼
- 불명확한 규정 질의에 대해 출처와 경고를 함께 제공함으로써 사용자의 불안 비용 감소 효과 기대

※ 발표자료 내 액체류 비중은 82.8%와 87.8% 표기가 혼재합니다. 본 문서에서는 막대그래프의 절대 건수 4,220,000건을 기준으로 해석했습니다.

## 5. 배운 점 및 향후 개선 방향

### 배운 점

- 규정 문서는 길지 않아도 표현 체계가 딱딱해 일반 사용자 질의와 직접 매칭되지 않는다는 점 확인
- RAG의 핵심은 더 큰 모델 투입보다 질의 표현 정규화와 라우팅 안정화에 있다는 점 확인
- 속도 문제는 생성 모델 교체보다 호출 횟수와 분기 구조 최적화가 더 큰 개선을 만든다는 점 확인
- 환각 문제는 후처리 문구보다 "검색 실패 시 확정 답변을 하지 않는 흐름 설계"가 더 중요하다는 점 확인

### 한계

- 현재 지원 범위가 한국(KR)·미국(US) 중심으로 제한된 점
- 국가 규정 중심 데이터이며, 항공사별 세부 예외 규정까지는 아직 미반영된 점
- 정답셋 기반 Top-k Recall, MRR, Faithfulness, Hallucination Rate 측정 체계 부재
- 미국 데이터의 경우 문서 단위 `evidence_url` 정밀도가 한국 데이터보다 낮은 점
- 이미지 기반 성분표, 배터리 라벨, 전자제품 스펙 판독이 불가능한 점

### 향후 개선 방향

- 다중 LLM 교차 검증 체계 도입
  - 생성 모델과 검증 모델 분리
  - 1차 답변 생성 후 2차 모델이 근거 문서 인용 적합성과 판정 일관성 검토
  - "규정 근거 존재 여부", "국가 일치 여부", "기내/위탁 판정 충돌 여부" 자동 감사 구조 도입

- 수치해석 기반 검색 최적화 고도화
  - 현재 경험적으로 설정된 `score <= 1.2` 임계값의 ROC/PR 기반 재보정
  - 정답셋 200~500문항 구축 후 Top-k Recall, Precision@k, Answer Accuracy 측정
  - 부트스트랩 신뢰구간 산출을 통한 검색 성능의 통계적 안정성 확보
  - BM25 + Vector Hybrid Retrieval 및 Cross-Encoder Reranker 도입 검토

- 데이터 스케일업 구조 고도화
  - 국가 추가 시 코드 수정 없이 데이터만 적재하는 확장형 파이프라인 유지
  - 항공사별 특수 규정 테이블 별도 계층화
  - 정책 개정일 기준 증분 업데이트 파이프라인 구축

- 멀티모달 OCR 도입
  - 성분표 촬영 기반 액체·식품 판별
  - 배터리 라벨 OCR 기반 Wh 자동 판독
  - 전자담배·보조배터리 등 회색지대 품목의 자동 분류 정확도 개선
