## 프로젝트 확장 배경

- 기존 텍스트 기반으로만 구현된 챗봇은 사용자의 입력단어에만 의존해 실제 물품의 형태나 규격을 반영하지 못함.
- 따라서 항공 보안 규정 검토의 신뢰도가 감소하고, 사용자가 물품의 정보를 정확히 입력하지 못하는 경우 규정 판단이 어려움.
- 이를 해결하기 위해 물품 이미지를 입력받아 물품 종류, 용량, 모델명까지 자동 인식하여 규정 판단까지 이어지는 Vision 파이프라인 구축 시도

## Vision 파이프라인

```
이미지 업로드
       ↓
┌─────────────────────┐
│  YOLOv8 물체 탐지    │  → 5가지 물품 분류 + Bounding Box
└─────────────────────┘
       ↓
┌─────────────────────┐
│  EasyOCR 텍스트 인식  │  → 크롭 영역 내 텍스트 추출
└─────────────────────┘
       ↓
┌─────────────────────┐
│  Regex V3 수치 파싱   │  → mAh / Wh / ml / g 등 수치 정밀 추출
│  + 오타 교정 로직     │     (7OOml → 700ml 자동 정정)
└─────────────────────┘
       ↓
┌─────────────────────┐
│  신뢰도 판정          │  → high / measured / low 등급 분류
│  + Fallback 로직     │     저신뢰도 시 사용자에게 수동 입력 유도
└─────────────────────┘
       ↓
┌─────────────────────┐
│  Vision-RAG 브릿지    │  → 인식 결과를 챗봇 슬롯에 자동 매핑
│  (process_vision_result) │  → 기존 RAG 파이프라인과 연동
└─────────────────────┘
```

## 프로젝트 과정 정리

### 데이터셋 확보

- [Roboflow](https://universe.roboflow.com/)에서 YOLO 학습용 데이터셋을 구했습니다.

| Split | 이미지 수 |
|-------|----------|
| Train | 4,031장 |
| Valid | 473장 |
| Test  | 205장 |

### 데이터셋 개요

- 항공 보안 검색대에서 가장 많이 적발되는 TOP5 물품을 선정하여 클래스를 나누었습니다.

| 클래스 | BBox 수 | 비고 |
|--------|---------|------|
| `power_bank` | 1,391 | 보조배터리 |
| `liquid_cosmetics` | 1,495 | 화장품·액체류 |
| `lighter` | 866 | 라이터 |
| `laptop` | 1,292 | 노트북 |
| `tablet` | 691 | 태블릿 |

### 데이터셋 학습 결과

- **사용 모델**: YOLOv8n 
- **전체 평균 mAP50**: **0.917**

| 클래스 | Precision | Recall | mAP50 |
|--------|-----------|--------|-------|
| power_bank | 0.945 | 0.920 | 0.952 |
| liquid_cosmetics | 0.890 | 0.885 | 0.910 |
| laptop / tablet | 0.920 | 0.910 | 0.935 |
| lighter | 0.850 | 0.820 | 0.840 |

## OCR 모델 선정 및 수치 추출

- EasyOCR, PaddleOCR, Tesseract 등 다양한 OCR 모델을 비교해 본 결과, EasyOCR이 데이터 수 대비 인식률에서 가장 준수한 성능을 보여줌
- 정규표현식 사용으로 규정에 필요한 수치 추출 과정에서 오타 교정, 범위 필터링, 다단위 지원, 한국어 패턴 대응 등 다양한 교정 진행  

| 기능 | 설명 |
|------|------|
| O/0 오타 교정 | `7OOml`, `7ooml` → `700ml` 자동 정정 |
| 수치 범위 필터링 | `5~3,000ml` 범위 제한으로 시리얼 번호 오인식 차단 |
| 다단위 지원 | mAh, Wh, V, ml, L, g, kg 등 모든 관련 단위 인식 |
| 한국어 패턴 | "밀리리터", "와트시" 등 한글 단위 표기 대응 |

### OCR 벤치마크 

| 버전 | 인식률 | 개선 사항 |
|------|--------|-----------|
| v1 (초기 Regex) | 31% | 단순 패턴 매칭 |
| v2 (오타 교정) | 62% | O/0 교정 + 컨텍스트 추론 |
| v3.1 (최종) | 65%+ | 엄격한 Fallback 가동, 환각 차단 |

### Confidence & Fallback

- OCR 인식 결과의 신뢰도를 3단계로 분류해 환각 현상 방지
- 사용자에게 "글씨 인식에 오류가 있을 수 있습니다. 용량을 직접 알려주세요." 라고 안내하여 서비스 신뢰성을 확보

1. 신뢰도 높음 : 수치 + 단위 모두 명확 → 자동 규정 판단
2. 신뢰도 보통 : 수치는 있으나 단위 불명확 → 추정치로 판정하되 주의 표시
3. 신뢰도 낮음 : 인식 실패 또는 모호 → 추정치 은닉 + 사용자에게 직접 입력 요청

## Vision-RAG 파이프라인 연결

- Vision 파이프라인의 분석 결과를 기존 RAG 챗봇의 슬롯에 연결 하는 로직 구축

| 입력 | 슬롯 매핑 | RAG 후속 처리 |
|------|-----------|---------------|
| 보조배터리 사진 | `item="power_bank"`, `quantity="20000mAh"` | 해당 용량의 기내/위탁 규정 자동 검색 |
| 화장품 사진 | `item="liquid_cosmetics"`, `quantity="50ml"` | 100ml 기준에 의한 즉시 판정 |
| 노트북 사진 | `item="laptop"`, `model="Galaxy Book"` | 리콜 DB 대조 후 안전 여부 안내 |
| 인식 실패 | `requires_manual_check=True` | 사용자에게 수동 입력 질문 |

## 한계 및 남은 과제

- 팀 프로젝트 이후 CV 분야 접목을 시도해보고자 도전했지만, 역시 공부해야할 점과 개선해야할 점이 많습니다.
- 아래는 프로젝트를 하는 과정에서 겪은 점을 정리해보았습니다.

| 한계 | 내용 |
|------|------|
| OCR 인식률 | 최종 65%로 실사용 수준에 미달. 글씨를 가까이 찍지 않으면 인식 실패 빈번 |
| 벤치마크 데이터 부족 | 테스트셋으로 직접 제품을 찍어 32장으로만 평가. 표본 수가 적어 통계적 신뢰도 낮음 |
| 지원 물품 제한 | 5가지 클래스만 탐지 가능. 실제 수하물 규정 대상 물품은 훨씬 다양함 |
| HEIC 포맷 | Mac 환경의 `sips` 명령어로 변환 처리했으나, 크로스 플랫폼 대응 미완성 |

- 역시 학습 & 테스트 데이터가 많아야하고, 더 많은 클래스와 다양한 촬영 조건의 학습 데이터 확보
- OCR 대신 기존 RAG 챗봇에서 사용했던 GPT-4o의 Vision API기능을 사용하는 방법도 존재한다고 한다. 그러나 본 프로젝트는 종료이후 제공받은 API키가 만료되어 사용할 수 없었다. 만약 대체 가능하다면 인식률을 크게 높일 수 있을 것으로 예상
- 이미지 파일의 경우 확장자가 플랫폼별로 달랐다. 하나의 플랫폼만 사용하지 않으니 다양성을 고려해볼 필요도 있다
