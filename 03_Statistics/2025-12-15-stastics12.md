## 12.1 주성분 분석

### 변수의 차원

'차원이 높다' -> 정보량이 많으니 얼핏 좋은 것처럼 생각됨
사실은 '쓸데없이 많기만'한 상황이 종종 발생.

데이터 특징을 유지하며 분석이나 결과 해석에 도움을 줄 수 있게끔 압축하는 것이 중요함.
이걸 차원 축소라고 한다.

#### 변수의 수를 줄이는 이유?
단순히 쓸데없는 데이터가 많아서 그런게 아님

1. 고차원 데이터 해석의 어려움이 있다 
    - 압축해서 2차원으로 평면에 나타내는게 가장 직관적이고 알기 쉽다
    - 다중회귀분석에서 설명변수끼리 강한 상관이 있는 다중공선성 -> 회귀계수 추정이 불안정해짐
2. '차원의 저주' 문제 
    - 변수가 많아진다 = 파라미터 수가 증가
    - 때문에 표본크기가 충분하지 않다면, 회귀계수를 올바르게 추정할 수 없음

### 주성분분석

차원축소의 대표적인 방법이 바로 주성분분석이다.
상관이 있는 변수끼리는 하나로 정리될 수 있다는 아이디어에 기반을 둔다

주성분 분석 새로운 축을 설정하고 그 축 위의 값으로 데이터를 새롭게 바라본다
어떻게 바라보냐면, 데이터 퍼짐이 가장 커지는 방향으로 설정함

그렇게 설정된 첫번째 축을 제1주성분(PC1)이라고 한다 그 다음 축을 구할 때, PC1에 수직이고 데이터 퍼짐이 커지는 방향을 구하면 그것이 제2주성분(PC2)가 된다

데이터 퍼짐이 커지는 방향을 찾는 이유는 각 축에 있는 정보가 데이터 퍼짐으로 나타난다고 생각하기 때문
그래서 원래 각 축에 있던 정보를 PC1으로 압축해 넣는다는 느낌

각 주성분이 가진 정보의 비율을 기여율, 제1부터 제k까지 전체 정보의 포함 비율을 누적기여율이라고 한다

원래 변수 사이에 강한 상관관계가 있을 수록 소수의 변수에 전체 정보 대부분이 있을 것으로 차원 축소가 잘될거라 기대함

### 주성분분석 결과
PC1은 다음과 같이 구함

$$ Z_1 = a_1 x_1 + a_2 x_2 + ... + a_M x_M $$

분산이 최대가 되도록 계수(a_1, a_2, ... a_M)를 정하고, PC2는 그때까지 얻은 주성분과 직교하도록 설정하며 새로운 축의 변수들끼리 서로 상관관계가 없게 만든다

주성분 분석은 데이터의 분산공분산행렬 또는 상관행렬을 바탕으로 계산하고, 계수는 행렬의 고유벡터, 기여율은 각 고유값을 고유값의 총합으로 나눈 값에 해당한다. 

주성분 부하량은 각 주성분과 원래 변수들 사이의 상관관계를 계산한 값이다. 기여율과 누적기여율은 주성분이 원본 데이터를 얼마나 잘 설명하는지를 확인한다. 

### 인자분석

1. 인자 분석(Factor Analysis)의 원리
인자 분석은 "관측되는 여러 변수들의 바탕에는 그것들을 지배하는 소수의 **공통 인자(Common Factor)**가 존재한다"는 아이디어에서 출발한다.

공통 인자: 여러 변수에 공통적으로 영향을 주는 잠재적인 원인이다.
독립 인자 (고유 인자): 개별 변수만이 가진 고유한 특성이나 오차를 의미한다.
분석의 목적: 겉으로 드러난 변수들(예: 국어, 수학 점수 등)을 통해 직접 측정할 수 없는 배후의 지능, 적성 같은 '인자'를 찾아내는 것이다.

2. 주성분 분석(PCA) vs 인자 분석(FA)의 차이
두 기법은 데이터를 축소한다는 점에서 비슷해 보이지만, 데이터를 바라보는 방향이 정반대이다.

| 구분 | 주성분 분석 (PCA) | 인자 분석 (FA) |
|---|---|---|
| 방향 | 변수들의 합 → 주성분 | 공통 인자 → 변수들 |
| 목적 | 정보의 손실을 최소화하며 차원 축소 | 변수 간 상관관계를 만드는 잠재 원인 파악 |
| 구조 | $Z = a_1x_1 + a_2x_2 + \dots$ | $x_1 = b_1(\text{인자}) + \text{오차}$ |

3. 결과 해석의 예시 (그림 12.1.4 관련)
이미지 왼쪽 페이지의 표를 통해 분석 결과를 해석하는 방법이 나와 있습니다.기여율: PC1과 PC2가 전체 정보의 약 **94%**를 설명하고 있어, 두 개의 주성분만으로도 충분히 데이터를 대표할 수 있음을 보여줍니다.의미 부여: * PC1: 수학, 과학 점수와 높은 상관관계 → **'이과적 능력'**으로 해석.PC2: 국어, 사회 점수와 높은 상관관계 → **'문과적 능력'**으로 해석.

## 12.2 기계학습 입문

1. 기계학습의 정의와 활용
기계학습은 컴퓨터가 데이터를 통해 학습하여 성능을 개선하는 기술로, 통계학과 밀접한 관련이 있다.
최근에는 통계학뿐만 아니라 비즈니스, 이미지 판정, 자율 주행 등 복잡한 데이터 분석 분야에서 널리 사용된다.
딥러닝(Deep Learning) 또한 기계학습의 한 종류로, 매우 높은 성능을 보이며 큰 주목을 받고 있다.

2. 기계학습의 3대 분류
기계학습은 크게 데이터의 성격(정답 유무)에 따라 세 가지로 나뉜다.
비지도 학습 (Unsupervised Learning): 정답 데이터가 없는 학습이다. 데이터의 숨겨진 구조를 찾거나 차원을 축소하며, **주성분 분석(PCA)**이나 군집 분석이 이에 해당한다.
지도 학습 (Supervised Learning): 정답(레이블)이 있는 데이터를 학습하여 예측 모델을 만든다. 통계학의 회귀 분석이 대표적인 형태이다.
강화 학습 (Reinforcement Learning): 에이전트가 환경과 상호작용하며 보상을 최대화하는 방향으로 행동을 학습하는 방식이다.

3. 통계학과 기계학습의 차이
두 기법은 도구를 공유하지만, 주안점이 다르다.

| 구분 | 통계학 | 기계학습 |
|---|---|---|
| 데이터 규모 | 상대적으로 작은 표본 데이터 | 대량의 데이터 (빅데이터) |
| 목적 | 데이터의 설명 및 해석 중시 | 높은 예측 성능 중시 |
| 특징 | 엄격한 가정 하에 원인 규명 | 복잡한 데이터 속 패턴 발견 |

## 12.3 비지도 학습

1. 통계학적 모델 vs 기계학습 모델의 차이
전통적인 통계학과 현대 기계학습의 시각 차이를 보충 설명한다.

통계학(선형회귀 등): 절편과 기울기를 가진 단순한 모델을 사용하여 변수 간의 관계를 명확히 해석하는 데 중점을 둔다.

기계학습: 모델이 복잡하여 해석은 어렵지만, 복잡한 현상에 대한 예측 성능이 매우 높다는 장점이 있다. (최근에는 해석력을 높이려는 연구도 진행 중이다.)

2. 비지도 학습(Unsupervised Learning)이란?
정의: 정답(레이블) 데이터가 없는 상태에서 데이터의 배후에 있는 구조를 파악하는 학습 방식이다.

주요 목적: * 비슷한 데이터들끼리 묶는 군집화.

고차원 데이터를 적은 수의 차원으로 줄이는 차원 축소.

3. 군집 분석 (Cluster Analysis)
비지도 학습의 대표 사례이다.

개념: 데이터 간의 유사성을 측정하여 비슷한 개체들을 하나의 그룹(군집)으로 묶는 방법이다.

예시: 옷 구매 고객 데이터를 분석할 때, '가격'과 '디자인'이라는 변수를 바탕으로 비슷한 구매 패턴을 보이는 고객들을 분류할 수 있다.

특징: "이 데이터는 어느 그룹에 속한다"라는 정답이 미리 주어지지 않으므로, 분석 결과는 데이터의 구조와 분석가의 설정(가정)에 따라 달라질 수 있다.

### 군집 분석 방법들

1. k-평균법 (k-means)가장 대표적인 비지도 학습 알고리즘 중 하나로, 데이터를 $k$개의 군집으로 나누는 방법이다.
과정: 먼저 $k$개의 중심점을 임의로 정하고, 각 데이터를 가장 가까운 중심점에 할당한다. 이후 군집 내 데이터들의 평균 위치로 중심점을 이동시키는 과정을 반복하며 최적의 군집을 찾아낸다.
특징: 계산이 빠르고 효율적이지만, 사용자가 군집 수($k$)를 직접 지정해야 한다는 특징이 있다.

2. 계층적 군집화 (Hierarchical Clustering)
데이터들을 순차적으로 가까운 것끼리 묶어가는 방식이다.

덴드로그램 (Dendrogram): 오른쪽 페이지 하단의 그림처럼 데이터가 결합되는 과정을 나무 모양의 계통도로 나타낸다.

특징: 이 그래프를 보고 분석가가 원하는 단계에서 선을 그어 군집의 개수를 유연하게 결정할 수 있다.

3. 거리의 정의와 결과의 주관성
군집 분석에서 '비슷하다'는 기준은 데이터 간의 거리로 측정한다.

거리를 계산하는 방식(유클리드 거리 등)에 따라 결과가 달라질 수 있으므로, 군집 분석 결과에는 분석가의 주관적인 판단이 개입될 수밖에 없다. 따라서 결과에 대한 논리적인 해석이 중요합니다.

4. PCA 이외의 차원 축소
앞서 배운 주성분 분석(PCA)은 변수 간의 선형 상관관계를 기반으로 차원을 축소한다. 하지만 최근 기계학습 분야에서는 데이터의 비선형적인 구조까지 파악하여 차원을 줄이는 더 복잡하고 강력한 방법들이 사용되고 있음을 언급하고 있다.

5. 비선형 차원 축소: t-SNE와 UMAP
선형적인 관계만 파악하는 PCA와 달리, 복잡한 비선형 관계를 가진 데이터를 시각화하는 방법들이다.

t-SNE: 고차원 데이터를 2차원 등으로 시각화할 때, 비슷한 데이터 구조를 잘 유지하며 군집을 명확하게 분리해 보여준다.

이미지 예시: 손글씨 숫자 데이터(MNIST)를 PCA로 축소하면 군집이 뭉쳐 보이지만, t-SNE를 사용하면 각 숫자별로 집합이 훨씬 명확하게 구분되는 것을 알 수 있다.

UMAP: t-SNE와 마찬가지로 비선형 관계를 다루며 시각화에 자주 쓰이는 또 다른 강력한 기법입니다.

## 12.4 지도학습

1. 지도 학습 (Supervised Learning)
정답이 있는 데이터를 바탕으로 규칙(함수)을 찾아내는 학습 방식이다.

정의: 설명변수($x$)와 반응변수($y$, 정답) 사이의 관계를 나타내는 함수를 구하는 과정이다.

학습의 원리: 모델이 내놓은 출력값과 실제 정답($y$) 사이의 차이를 **손실 함수(Loss Function)**로 정의하고, 이 차이를 최소화하도록 모델의 파라미터를 업데이트한다.

통계적 회귀와의 공통점 및 차이점:
공통점: 최소제곱법처럼 오차를 최소화하는 파라미터를 구한다는 점은 같다.

차이점: 통계학은 관계 해석에 집중하는 반면, 기계학습은 대량의 데이터와 복잡한 함수를 사용하여 예측 성능을 극대화하고 과적합을 피하기 위한 정칙화(Regularization) 기법 등을 적극적으로 활용한다.

2. 회귀(Regression)와 분류(Classification)
지도 학습은 예측하고자 하는 타겟 데이터(반응변수 $y$)의 형태에 따라 나뉜다.

회귀: $y$가 연속적인 **수치(양적 변수)**일 때 사용한다. (예: 주가 예측, 온도 예측)

분류: $y$가 **카테고리(질적 변수)**일 때 사용한다. 두 가지 중 하나를 고르는 이진 분류부터 여러 항목을 맞히는 다중 클래스 분류까지 가능하다. (예: 스팸 메일 여부, 숫자 판독)

3. 예측력과 교차 검증 (Cross Validation)
학습에 사용하지 않은 미지의 데이터를 얼마나 잘 맞히는지가 기계학습의 핵심이다.

필요성: 학습 데이터만 너무 잘 맞히는 모델은 실전에서 쓸모가 없을 수 있다. 따라서 데이터를 학습용과 검증용으로 나누어 테스트한다.

K-분할 교차 검증: 데이터를 $K$개로 나눈 뒤, 그중 하나를 검증 데이터로 쓰고 나머지를 학습 데이터로 사용하는 과정을 $K$번 반복하여 평균 성능을 낸다.

LOOCV (Leave-one-out): 데이터 하나만 검증용으로 남기고 나머지를 모두 학습에 쓰는 극단적인 형태의 교차 검증이다.

4. 과대적합 (Overfitting)과 정칙화 (Regularization)
과대적합: 모델이 학습 데이터의 사소한 노이즈까지 다 외워버려서, 정작 새로운 데이터(검증 데이터)에 대해서는 예측력이 떨어지는 현상이다.

정칙화: 모델이 너무 복잡해지지 않도록 손실 함수에 제약을 거는 방법이다. 복잡도를 조절하여 과대적합을 방지하고 범용적인 성능을 높인다.

### 예측 성능 평가

1. 혼동 행렬 (Confusion Matrix)
예측값과 실제 정답의 조합에 따라 4가지 케이스로 분류한다.

진양성 (True Positive, TP): 양성(Positive)이라고 예측했는데 실제로 양성인 경우.

위양성 (False Positive, FP): 양성이라고 예측했는데 실제로 음성인 경우 (1종 오류).

위음성 (False Negative, FN): 음성이라고 예측했는데 실제로 양성인 경우 (2종 오류).

진음성 (True Negative, TN): 음성이라고 예측했는데 실제로 음성인 경우.

2. 주요 성능 평가 지표
혼동 행렬의 수치들을 조합하여 모델의 성능을 다각도로 평가한다.

정확도 (Accuracy): 전체 데이터 중 맞게 예측한 비율$$\frac{TP + TN}{TP + FP + FN + TN}$$

민감도 (Sensitivity / Recall): 실제 양성인 것들 중 양성으로 잘 맞힌 비율$$\frac{TP}{TP + FN}$$

특이도 (Specificity): 실제 음성인 것들 중 음성으로 잘 맞힌 비율$$\frac{TN}{FP + TN}$$

정밀도 (Precision / Positive Predictive Value): 양성이라고 예측한 것들 중 실제 양성인 비율$$\frac{TP}{TP + FP}$$

3. 민감도와 특이도의 관계 (Trade-off)
일반적으로 민감도가 높아지면 특이도는 낮아지는 상충 관계가 있다.

예를 들어, 모든 데이터를 무조건 '양성'으로 판정하면 민감도는 100%가 되지만, 특이도는 0%가 되어 모델로서의 가치가 사라진다.

이 두 지표를 동시에 고려하여 모델의 성능을 시각화한 것이 다음에 나올 ROC 곡선과 AUC이다.

### 분류 및 회귀의 성능 측정

1. 분류: ROC 곡선과 AUC

ROC 곡선: 가로축을 위양성률(1-특이도), 세로축을 민감도로 하여 모델의 성능을 그래프로 나타낸 것이다. 곡선이 왼쪽 위 구석에 가까울수록 성능이 우수하다.

AUC (Area Under the Curve): ROC 곡선 아래의 면적 수치이다. 1에 가까울수록 완벽한 모델이며, 0.5라면 무작위 추측과 다를 바 없는 최저 성능을 의미한다.

2. 회귀: 오차 측정 지표

반응변수가 수치형인 경우, 실제값($y$)과 예측값($\hat{y}$)의 차이를 계산한다.

MSE (평균제곱오차): 오차의 제곱을 평균한 값이다.

RMSE (제곱근 평균제곱오차): MSE에 루트를 씌워 단위의 스케일을 맞춘 지표이다.

MAE (평균절대오차): 오차의 절댓값을 평균한 것으로, 이상치(outlier)의 영향에 더 강건(robust)하다.

$R^2$ (결정계수): 모델이 데이터의 분산을 얼마나 잘 설명하는지 나타내며, 1에 가까울수록 좋다.

### 주요 기계학습 알고리즘

1. 로지스틱 회귀 (Logistic Regression)

이름은 회귀지만 실제로는 분류 모델이다. 데이터가 특정 클래스에 속할 확률을 0과 1 사이의 값으로 출력한다. 선형적인 경계로 클래스를 나눈다.

2. 결정 트리 & 랜덤 포레스트

결정 트리 (Decision Tree): 스무고개처럼 조건 분기($x > 10$ 등)를 통해 데이터를 분류한다. 직관적이지만 과대적합되기 쉽다.

랜덤 포레스트 (Random Forest): 여러 개의 결정 트리를 만들어 결과를 종합(앙상블)함으로써 예측 성능을 높이고 과대적합을 방지한다.

3. SVM & 신경망 (Deep Learning)

SVM (서포트 벡터 머신): 데이터 군집 간의 거리를 최대화하는 경계면을 찾는다. 비선형 분류도 가능하다.

신경망 (Neural Network): 뇌의 뉴런 구조를 모방한 모델이다.

딥러닝: 신경망을 여러 층(은닉층)으로 쌓은 형태이다.

오차역전파법: 출력에서 발생한 오차를 거꾸로 전파하며 가중치를 수정하여 학습한다. 복잡한 패턴 파악에 강력하지만 '블랙박스'처럼 내부 해석이 어렵다는 단점이 있다.

### 선형 분리가 가능한 경우

데이터가 직선 하나로 깔끔하게 나뉠 수 있는 상황이다.

로지스틱 회귀: 하나의 직선으로 영역을 나눈다. 가장 단순하고 명확하다.

다른 모델들: 결정 트리, SVM, 신경망 모두 직선에 가까운 형태로 무난하게 데이터를 분류해냅니다.

2. 선형 분리가 불가능한 경우 (그림 12.4.6)
데이터가 복잡하게 뒤섞여 있어 직선 하나로는 해결이 안 되는 상황입니다.

로지스틱 회귀 (한계): 여전히 직선으로만 나누려 하기 때문에 정답률이 50% 수준으로 뚝 떨어집니다. 즉, 복잡한 비선형 데이터에는 적합하지 않습니다.

결정 트리: 가로축과 세로축에 평행한 직선들을 조합하여 '계단 형태'로 구역을 나눕니다.

SVM & 신경망: 부드러운 곡선 형태의 경계를 만들어 복잡한 데이터도 매우 높은 정답률(95% 이상)로 분류해냅니다.
