# Narrative Plan

## Audience

- Data analyst / data scientist hiring reviewers
- Semiconductor process analytics or monitoring-oriented interview audience

## Objective

- Present this project as an anomaly monitoring and rule-tuning portfolio, not as a trading strategy deck
- Highlight data audit, feature review, baseline comparison, false alert control, and operational storytelling

## Narrative Arc

1. Reframe crypto minute data as a monitoring problem
2. Show why raw data quality had to be addressed first
3. Summarize feature review and baseline comparison
4. Show that rule tuning reduced false alert burden
5. Translate the project into semiconductor monitoring language

## Slide List

1. Cover
2. Why monitoring, not trading
3. Data audit and risk
4. Feature review
5. Baseline comparison
6. Rule tuning logic
7. Final test improvement
8. Daily operations KPI
9. Semiconductor translation
10. Closing

## Source Plan

- Local notebooks in `notebooks/` and output CSV files in `outputs/`
- Final KPI values from notebook 04 and 05 outputs
- Presentation structure derived from `docs/PRESENTATION_OUTLINE.md`

## Visual System

- Clean editorial layout
- Light neutral background
- Green accent for detection/monitoring signal
- Card-based information hierarchy
- Large readable Korean text

## Image Plan

- Draft deck prioritizes editable layout and presentation flow
- Use deck-native geometry and the template plate fallback for this first draft
- If needed later, replace fallback plates with slide-specific generated background plates

## Editability Plan

- All visible text remains editable PowerPoint text
- Core content uses title blocks, cards, and metric tiles
- No chart screenshots or flattened text images
