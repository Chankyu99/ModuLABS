## 0-2. 결측 처리 방침

> 💬 **Q. 결측률에 대해 어떻게 생각해? 이걸 처리해야겠지?**

### 결측 패턴 분석 결과

| 패턴 | 건수 | 비중 |
|:---|---:|:---:|
| Actor1 ✅ Actor2 ✅ | 16,037,771 | 60.3% |
| Actor1 ✅ Actor2 ❌ | 6,331,915 | 23.8% |
| Actor1 ❌ Actor2 ✅ | 4,222,493 | 15.9% |
| Actor1 ❌ Actor2 ❌ | **0** | **0%** |

**양쪽 다 결측인 경우가 0건**. 항상 최소 한쪽은 국가 코드가 존재한다.

결측인 Actor의 Name을 확인하면:
- PRESIDENT, MILITARY, POLICE, GOVERNMENT, TERRORIST, PROTESTER, MEDIA...
- → **국가가 아니라 역할명**. "이란의 대통령"처럼 국적이 기사에 명시되지 않은 경우

### 결론: **결측 처리 불필요**

| 이유 | 설명 |
|:---|:---|
| OR 필터가 자연 처리 | `Actor1CountryCode IN ('IRN','USA','ISR') OR Actor2CountryCode IN (...)` — 한쪽만 있으면 작동 |
| 대입 근거 없음 | "MILITARY → 이란 군" 식의 추정은 거짓 데이터를 만듦 |
| 데이터 손실 0건 | 양쪽 다 비는 경우가 없으므로 필터링에서 놓치는 건 없음 |

## 0-3. ActionGeo_Type = 4 (도시) vs Type = 1 (국가)

> 💬 **Q. ActionGeo_Type = 4만 쓸지, Type = 1도 포함할지가 왜 중요한 건지 설명해줘.**

| Type | 비중 | 좌표 예시 | 위성 촬영 |
|:---|:---:|:---|:---:|
| **Type 4 (도시)** | 70.6% | Tehran (35.69, 51.42) — 반경 ~10km | ✅ 촬영 가능 |
| Type 1 (국가) | 28.6% | Iran (32.00, 53.00) — 164만 km² 중심 | ❌ 좌표 무의미 |

같은 "이란에서 군사 충돌 발생" 뉴스라도:
- Type 4: "테헤란에서" → 위성을 테헤란으로 ✅
- Type 1: "이란에서" → 이란 어디? 164만 km² 중 어디를 찍지? ❌

**→ 용도별 분리 사용:**
- 전이 확률, 여론 과열 지수 계산 → **전체 데이터** (Type 무관)
- ROI 선정, 촬영 좌표 결정 → **Type = 4만**

### 핵심 발견

1. **Root 19의 자기 상관이 극히 높음** (lag1 = 0.879)
   - 교전이 시작되면 다음 날에도 88%의 확률로 유사한 수준의 교전이 지속
   - 골든타임 3일 설정의 통계적 근거

2. **Root 18 (폭행)이 가장 강력한 선행 지표** (lag0 = 0.859)
   - 비정규 폭력(테러, 납치, 폭탄)이 발생한 날, 같은 날 교전도 높은 확률로 동반
   - 폭행 → 교전의 인과가 아니라 **동시 발생(co-occurrence)** 패턴

3. **Root 15 (군사태세)는 시간차 선행 지표**
   - lag0 = 0.661이지만 lag3에서도 0.499로 상당히 유지
   - 군사력 배치 → 2~3일 후 교전으로 이어지는 패턴

4. **Root 17 (강압)은 상관이 가장 낮음** (lag3 = 0.326)
   - 체포, 재산 압수 등은 교전과 직접 연결되지 않음
   - 포함은 하되, 트리거 판정에서 가중치는 가장 낮게

> 💬 **Q. 19→19, 18→19, 15→19 이런 순으로 선행지표로 보면 되는거지?**

| 패턴 | 상관계수 (lag1) | 성격 | 의미 |
|:---|:---:|:---|:---|

| **18→19** | 0.782 | **동반 발생** | 테러/폭행이 나면 교전도 같이 터진다 |

위성 촬영 스케줄링 관점에서:
- **15가 뜨면 → "준비하라" (2~3일 여유)**

- **19가 뜨면 → "계속 찍어라" (3일간 지속)**

## 2. AvgTone — 같은 코드 안에서 심각도를 구분하는 유일한 지표

Root 19(교전) 283만 건은 전부 GoldsteinScale = -10.0이지만, AvgTone은 **-39에서 +17까지 분포**합니다.

### AvgTone 구간별 NumMentions (= 사건 규모)

| AvgTone 구간 | 이벤트 수 | 비중 | **평균 NumMentions** | 해석 |
|:---|---:|:---:|---:|:---|
| ≤ -15 (극단 부정) | 10,541 | 0.4% | **17.6** | 극소수, 극도의 부정 보도 |
| -15 ~ -10 (매우 부정) | 173,468 | 6.1% | **19.2** | 심각한 사건 |
| **-10 ~ -5 (부정)** | **1,242,421** | **43.9%** | **22.4** | **가장 많고, 언급량 최대** |
| -5 ~ 0 (약한 부정) | 1,006,638 | 35.6% | 12.3 | 상대적으로 경미 |
| \> 0 (중립/긍정) | 398,063 | 14.1% | 11.6 | 맥락적 언급 |

> **AvgTone이 -10 ~ -5 구간일 때 NumMentions가 가장 높음(22.4)**
>
> 직관과 살짝 다른 이유: AvgTone ≤ -15인 극단 부정은 소규모 사건의 매우 감정적 보도이고, -10 ~ -5인 구간이 실제로 **대형 사건이 광범위하게 보도된** 경우

### AvgTone의 올바른 활용법

```
❌ 잘못: AvgTone이 -39면 가장 심각한 사건
✅ 올바름: AvgTone이 -10~-5이면서 NumMentions가 높으면 대형 사건
```

## 5. 3가지 지표의 올바른 활용 방식

```
GoldsteinScale  →  1단계 필터: "갈등인가 협력인가?"
                   (< -7: 심각한 갈등 = 우리 데이터)

AvgTone         →  2단계 심각도: "같은 교전인데 얼마나 심각한가?"
                   (-10~-5 구간 + NumMentions 높으면 = 대형 사건)

NumMentions     →  3단계 중요도: "전 세계가 주목하는 사건인가?"
                   (상위 1%면 역사적 사건)
```

### 파이프라인 적용

| 단계 | 지표 | 용도 |
|:---|:---|:---|
| 데이터 필터링 | GoldsteinScale < -7 | 분석 범위 설정 |
| 전이 확률 계산 | EventRootCode | 1차 판정 (관심/주의/경계/심각) |
| 여론 과열 지수 | **AvgTone 주간 이동평균** | 2차 판정 보조 |
| 사건 크기 판단 | **NumMentions** | 촬영 우선순위 가중 |

**"예측하지 못한다" ≠ "관련없다"**. 시간 스케일의 차이:

| 관계 유형 | 시간 스케일 | 예시 |
|:---|:---:|:---|
| **역사적 인과** | 수개월~수년 | 하마스 기습(2023.10) → 이란 보복(2024.04) → 전쟁(2026.02) |
| **통계적 선행** | 수일 | "오늘 ISR-PAL 100건 → 내일 Triad 50건" ← 이것은 불가 |

비유: 담배가 폐암의 원인이지만, "오늘 담배를 피웠으니 내일 폐암이 생긴다"고 예측할 수는 없는 것과 같다. **원인이지만 일별 예측 지표는 아니다.**

## 결론: 제외하지 말고, 구분해서 사용

| 용도 | 사용할 데이터 | 이유 |
|:---|:---|:---|
| **촬영 트리거** | Triad만 | ISR-PAL은 상관 r=0.17 (무관) |
| **배경 레벨 판정** | Triad + ISR-PAL + Proxy | 전체 중동 긴장도 파악 |
| **ROI 좌표** | Phase에 따라 분리 | P2: Gaza 중심 → P5: Tehran 중심 |

## Proxy(대리 세력)란?

> 💬 **Q. Proxy의 의미가 무엇인가?**

이란이 직접 싸우는 대신 중동 곳곳에서 **이란을 대신해 싸워주는 무장 세력**을 Proxy(대리 세력) 또는 **"저항의 축(Axis of Resistance)"**이라 부른다:

```
이란 (본체)
  ├── 헤즈볼라 (레바논) ──→ 이스라엘 북부
  ├── 하마스 (가자) ──→ 이스라엘 남부
  ├── 후티 (예멘) ──→ 홍해/사우디
  └── 이라크 민병대 ──→ 미군 기지
```

2026년 전쟁에서도 이란이 직접 공격하면서 동시에 헤즈볼라가 미사일 발사를 재개하고, 이라크 민병대가 미군 기지를 공격. Proxy 데이터는 이란 직접 공격 전에 **대리 세력이 먼저 움직이는 패턴**이 있을 수 있어 향후 선행 지표로 검토할 가치가 있다.

## 파이프라인 데이터 구조

> 💬 **Q. 이벤트 수가 2,452건이라는 것은 결국 2,660만 건에서 겨우 2,452건만 사용한다는 뜻인가?**

아니다. **용도별로 다른 범위의 데이터를 사용한다:**

```
2,660만 건 (전체)
  └─ 462만 건 (Root 15/17/18/19/20)     ← 전이확률, 여론과열 계산
       └─ 10만 건 (Triad: IRN↔USA↔ISR)  ← 트리거 판정
            └─ 2,452건 (핵시설 도시)      ← 최종 촬영 좌표 출력
```

| 단계 | 데이터 규모 | 용도 |
|:---|:---:|:---|
| **전체 2,660만** | 입력 | 전체 중동 긴장도 배경 모니터링 |
| **462만** (QC4 Root 15~20) | 분석 | 전이 확률, 여론 과열 지수 계산 |
| **10만** (Triad) | 트리거 | "지금 위기인가?" 판단 |
| **2,452** (핵시설 도시) | **출력** | "어디를 찍을까?" 결정 |

비유: 기상청이 전국 1,000개 관측소 데이터를 분석해서 → "내일 서울에 폭우" 한 줄을 출력하는 것과 같다. **2,660만 건 전부가 파이프라인에 입력되고, 핵시설 좌표는 최종 출력**이다.
