# 7. 군집화

본 문서는 K-평균, 평균 이동, GMM, DBSCAN 등의 다양한 군집화 알고리즘에 대한 핵심 개념과 실습 코드 실행 결과를 바탕으로 작성된 정리 노트입니다.

---

## 7-1. K-평균 알고리즘 이해

군집 중심점(Centroid)이라는 특정한 임의의 지점을 선택해, 그 중심에 가장 가까운 포인트들을 선택하며 데이터를 그룹화하는 기법입니다.

- **동작 과정 (6단계)**
1. **중심점 설정**: 군집화할 개수($k$)만큼 임의의 위치에 중심점을 배치합니다.
2. **데이터 할당**: 각 데이터는 가장 가까운 중심점에 소속됩니다.
3. **중심점 이동**: 소속된 데이터들의 평균 위치로 중심점을 이동시킵니다.
4. **재할당**: 중심점이 이동했으므로, 다시 각 데이터를 가장 가까운 중심점으로 변경합니다.
5. **반복**: 데이터의 소속이 변경되면 다시 3번 과정(평균 지점으로 이동)을 수행합니다.
6. **종료**: 중심점을 이동시켜도 데이터의 소속 변경이 없으면 군집화를 완료합니다.

- **K-평균의 장단점**
| 구분 | 주요 내용 |
| --- | --- |
| **장점** | • 가장 일반적으로 사용되는 쉽고 간결한 알고리즘입니다. |
| **단점** | • 거리 기반이라 속성(피처)이 너무 많으면 정확도가 떨어집니다 (PCA 차원 축소 필요).<br>• 반복 횟수가 많을 경우 수행 시간이 느려집니다.<br>• 몇 개의 군집($k$)을 설정해야 할지 사전에 결정하기 어렵습니다. |

### 사이킷런(Scikit-learn) KMeans 클래스 활용

파이썬의 `sklearn.cluster.KMeans`를 사용할 때 알아야 할 주요 파라미터와 속성입니다.

- **주요 파라미터**
  - `n_clusters`: 군집화할 개수(중심점의 개수)를 지정합니다.
  - `init`: 초기에 중심점 좌표를 설정하는 방식입니다. 보통 `k-means++` 방식을 사용하여 무작위 배치의 비효율성을 개선합니다.
  - `max_iter`: 최대 반복 횟수입니다. 이 횟수 이전에 이동이 없으면 조기 종료됩니다.

- **결과 확인을 위한 속성**
  - `labels_`: 각 데이터 포인트가 최종적으로 어느 군집 중심점에 속했는지 알려주는 레이블입니다.
  - `cluster_centers_`: 각 군집 중심점의 좌표(Shape는 [군집 개수, 피처 개수])입니다. 이를 통해 중심점 위치를 시각화할 수 있습니다.

### 붓꽃(Iris) 데이터 세트를 활용한 K-평균 군집화

1. **데이터 준비 및 K-평균 모델 학습**
붓꽃 데이터(꽃받침, 꽃잎 길이/너비 4차원)를 로드하고 군집 개수 `n_clusters=3`으로 설정하여 `fit()`을 수행합니다.

2. **군집화 결과 평가**
원래 품종(target)과 모델이 스스로 구분한 결과(cluster)를 비교해 봅니다.

```text

0       1          50
1       0          47
        2           3
2       0          14
        2          36
```
- **Target 0(Setosa)**: 1번 군집으로 50개 데이터 완벽하게 분리되었습니다.
- **Target 1(Versicolor), 2(Virginica)**: 상당히 잘 분리되었으나, 일부 데이터가 서로의 군집에 섞여 있는 것을 확인할 수 있습니다. 군집화는 정답 레이블을 모르는 상태에서 거리를 기반으로만 묶기 때문에 실제 정답과 100% 일치하기는 매우 어렵습니다.

4차원의 속성을 시각화할 수 없으므로, PCA를 통해 2개(pca_x, pca_y) 주성분으로 압축 후 산점도를 그립니다.

- 네모 마커: 다른 군집과 매우 명확히 분리됨.

![CLUS-3](Img/CLUS-3.png)

## 7-2. 군집 평가 (Cluster Evaluation) - 실루엣 분석

### 군집 평가의 필요성
대부분의 실전 군집화 데이터는 정답 레이블(Target)이 없습니다. 분류 알고리즘처럼 예측 정확도를 계산할 수 없으므로, 데이터 내 분리도를 측정하는 **실루엣 분석(Silhouette Analysis)**을 지표로 활용합니다.

### 실루엣 분석의 핵심
실루엣 분석은 각 군집 간의 거리가 얼마나 효율적으로 분리되어 있는지를 나타내는 계수입니다.

$$s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))}$$

- **$a(i)$ (응집도)**: 자신이 속한 군집 내 다른 데이터와의 평균 거리 (작을수록 결속력이 좋음)
- **$b(i)$ (분리도)**: 가장 가까운 다른 군집과의 평균 거리 (클수록 분리가 잘 됨)

- **계수 범위 (-1 ~ 1)**: 1에 가까울수록 근처 군집과 잘 떨어져 정상적으로 군집화된 것이고, 0에 가까우면 경계에 걸쳐 있는 것이며, 음수(-)는 아예 다른 군집에 잘못 할당되었음을 의미합니다.

### 실습: 붓꽃 데이터의 실루엣 분석 결과

```text
붓꽃 데이터셋 Silhouette Analysis Score (전체 평균): 0.551

개별 군집별 평균 실루엣 계수:
cluster 0: 0.422
cluster 1: 0.798
cluster 2: 0.437
```
1번 군집은 홀로 0.8에 가깝게 매우 잘 뭉쳐있고 떨어져 있으나, 0번과 2번 군집은 서로 거리가 다소 겹쳐 계수가 0.4대로 낮아졌습니다. **단순히 전체 평균이 높다고 좋은 것이 아니라 개별 군집의 점수 편차가 작고 두루두루 높아야 훌륭한 군집화**입니다.

### 실루엣 시각화 차트를 통한 최적의 K 찾기
`make_blobs`를 활용해 각각 2, 3, 4, 5개의 중심점을 설정하고 실루엣 분석의 분포 시각화(칼 모양의 차트)를 그리면 최적의 매개변수를 찾기 수월합니다.

![CLUS-4](Img/CLUS-4.png)
- **k=2일 때**: 군집 하나가 유난히 거대한 비율을 차지하며 편차가 큽니다. 전체 평균은 0.7 수준으로 높으나 좋은 분리라 보기 힘듭니다.
- **k=4일 때**: 전체 평균 점수는 k=2 때보다 약간 낮아도, 모든 군집 데이터가 평균 선(빨간 점선)을 무난히 넘기며 **가장 고르고 균형 있게** 분리되었습니다. 이상적인 개수로 볼 수 있습니다.

![CLUS-5](Img/CLUS-5.png)
- 동일한 검증을 붓꽃 데이터에 수행했을 때, k=2인 경우가 가장 이상적인 평균 실루엣 값(0.68)과 분포를 보여줍니다. 군집 개수가 커질수록 평균 점수도 떨어지고 찌그러지는 군집이 속출합니다. 

> **주의사항**: 실루엣 분석은 $O(N^2)$ 복잡도를 가져, 데이터 양이 무수히 많아지면 연산 시간이 급격히 증가하고 메모리 한계에 도달하므로 샘플링 기법을 혼용해야 합니다.

## 7-3. 평균 이동 (Mean Shift)

K-평균이 "무작위 중심점에서의 평균 거리"를 따진다면, 평균 이동은 **"가장 밀도가 높은 곳으로 중심을 잡자"**라는 원리로 움직입니다.
**KDE(커널 밀도 추정)**를 바탕으로 확률 밀도 함수의 피크가 위치한 지점을 지속적으로 추적하며 군집을 스스로 정합니다.

### 특징 및 파라미터 튜닝
K-평균과 달리 군집의 개수($k$)를 미리 정할 필요가 없습니다. 대신 알고리즘 자체를 지배하는 **대역폭(Bandwidth, $h$)** 파라미터를 갖습니다.

- $h$가 작을 때: 데이터의 굴곡을 너무 세밀하게 반영해 뾰족한 KDE를 만들어 과적합(수많은 군집) 유발.
- $h$가 클 때: 전체적으로 너무 완만하게 뭉뚱그려 과소적합 유발.

대역폭에 민감하게 결과가 변하기 때문에 사이킷런은 최적의 대역폭을 연산해 주는 `estimate_bandwidth()` 함수를 지원합니다.

```python
from sklearn.cluster import estimate_bandwidth
# 최적의 대역폭 자동 계산
bandwidth = estimate_bandwidth(X)
```
```text
bandwidth 값: 1.816
```
이를 반영하여 Mean Shift를 적용한 가상 데이터 산점도 시각화 결과, 원래 의도했던 3개의 군집으로 매우 명확하고 깔끔하게 묶어냅니다.

![CLUS-6](Img/CLUS-6.png)

> **요약**: 트래킹, 영상 이미지 세그멘테이션에서 매우 훌륭하나 군집의 연산량이 너무 무겁다는 한계가 있습니다. 가벼운 데이터 환경에서 데이터 개별 형태를 가정하지 않고 유연하게 쓰고 싶을 때 가장 선호됩니다.

## 7-4. GMM (Gaussian Mixture Model)

GMM은 **"전체 데이터 분포는 여러 개의 정규 분포(가우시안 분포)가 혼합된 모형일 것이다"**라고 가정하고 각 군집의 평균과 분산을 추정해 나가는 알고리즘입니다. 보통 EM(기댓값 최대화) 방식을 토대로 동작합니다.

데이터의 중심점과의 거리가 기준이었던 기존 방법론과 다르게 해당 분포도 포물선 안에 소속될 '확률' 기반으로 연산한다는 점이 매우 큰 차이입니다.

### K-평균 vs GMM의 비교 (붓꽃 데이터)

```text
### GMM Clustering ###
target  gmm_cluster
0       1              50
1       0              45
        2               5
2       0               0
        2              50

### KMeans Clustering ###
target  kmeans_cluster
0       1                 50
1       0                 47
        2                  3
2       0                 14
        2                 36
```
군집 경계가 모호했던 Target 1과 Target 2에 대하여, **GMM 모델이 KMeans 모델보다 붓꽃 원본 카테고리에 훨씬 가깝게 예측 분리**를 해냈습니다. GMM 예측력이 상대적으로 높다는 점을 보여줍니다.

### 데이터 분포 모양에 따른 GMM의 진면목

K-평균은 거리 중심이라 모든 군집을 **원형(Circular)**으로만 파악합니다. 반면 GMM은 가우시안 분포 확률이므로 **타원형(Elliptical)**으로 길게 늘어진 데이터에 탁월한 대응력을 가집니다.

가상 데이터를 타원형으로 일부러 길쭉하게 변형하여 `make_blobs` 데이터 세트를 만들어 봅니다.

![CLUS-7](Img/CLUS-7.png)

이를 기존의 **K-평균**으로 분리해 보겠습니다.
![CLUS-8](Img/CLUS-8.png)
거리 기준이다 보니 대각선 형태를 인지하지 못하고 중앙을 수평/수직으로 잘라버려 서로 다른 군집 레이블이 뒤섞이는 대참사가 일어납니다.

이를 **GMM**으로 분리해 보겠습니다.
![CLUS-9](Img/CLUS-9.png)
데이터가 뻗어나가는 방향(분산)을 유연하게 담아내 타원 형태 그대로 놀라운 성능으로 분리해냈습니다.
단점으로는 확률을 모델링하는 수학적 연산 때문에 GMM 역시 학습 수행 시간이 대략적으로 수 배 이상 더 소요됩니다.

## 7-5. DBSCAN (밀도 기반 군집화)

DBSCAN은 이름(Density-Based Spatial Clustering of Applications with Noise) 그대로 기하학적으로 일관성이 없거나 매우 정교한 밀도를 갖는 공간에 포진된 데이터를 노이즈(이상치)와 함께 구분할 수 있는 최고 수준의 유연한 도구입니다.

- **두 가지 핵심 조절 파라미터**
  - **`eps` (입실론)**: 나를 중심으로 하는 원의 반경(주변 영역 크기).
  - **`min_samples`**: 내 곁 반경 내에 있어야 군집의 본선, 즉 통과 기준을 만족하는 최소 이웃 데이터 개수.

- **처리 과정**
  조건을 충족하는 '핵심 포인트'를 중심으로 주변 영역을 침범하며 점차 '경계 포인트'들까지 자기 편으로 만듭니다. 그 어디에도 낄 구석이 없는 멀리 떨어진 소수의 외곽 데이터는 -1이라는 고유 번호표를 붙여 완벽하게 **노이즈(Noise) 데이터**로 취급해버립니다.

### 파라미터 조절 시각화 비교 (붓꽃 데이터)

- **기본 설정 (`eps=0.6, min_samples=8`)**
![CLUS-10](Img/CLUS-10.png)
가운데 크게 밀집된 구간이 전부 파란 십자가 형태로 단일 군집(Cluster 1) 덩어리가 되었습니다. (K-평균에서 2개의 품종이 살짝 겹쳐있던 구역을 DBSCAN은 "원래 하나같이 밀집된 동네구나"라고 본 것). 멀리 튕겨 나간 작은 별포인트(-1)는 노이즈 데이터로 식별되었습니다.

- **`eps=0.8` 로 영역 증가 (`min_samples=8`)**
![CLUS-11](Img/CLUS-11.png)
반경을 넓히니 이전에는 서로 닿지 않아 노이즈 취급받던 외곽 데이터들이 모두 파란 군집 안으로 수렴되었습니다. 노이즈가 단 하나도 검출되지 않습니다.

- **`min_samples=16` 로 최소 개수 기준 강화 (`eps=0.6`)**
![CLUS-12](Img/CLUS-12.png)

> **최종 요약**
DBSCAN은 군집 개수를 미리 몰라도 될 뿐더러 노이즈와 기하학적 형태에 가장 우수한 만능입니다. 다만, 데이터 내부의 근원적 밀도가 균일하지 않고 들쭉날쭉하다면 군집을 놓치는 곳이 발생해 파라미터(`eps`, `min_samples`)의 미세 튜닝(노가다)이 동반되어야만 제 성능을 발휘합니다.
