## 1.2 데이터와 데이터베이스의 개념

### 데이터 = **모든 기록들**

### 데이터베이스 = **데이터가 모여져 있는 공간**, **구조적인 방식으로 관리되는 데이터들의 집합**

> 특징 
>1. 실시간 접근성 : 사용자 질의에 대해 즉시 처리 & 응답
>2. 지속적인 변화 : 데이터의 삽입, 삭제, 갱신으로 인해 항상 최신 상태 유지
>3. 동시공유 : 여러 사용자가 동시에 접근하여 데이터 사용 가능
>4. 내용에 대한 참고 : 데이터의 일관성과 정확성을 유지하여 신뢰할 수 있는 정보 제공

### 관계형 데이터베이스(Relational Database) : 2차원 테이블(행,열)로 구성되어 데이터를 정의하고 설명하는 모델
- 구조화된 데이터는 SQL 쿼리문을 작성해 조회 가능
- 일관성이 있고, 안정적이며 신뢰성이 있다.

## 1.3 테이블

- 관계형 데이터베이스에서 자료의 구조를 2차원의 표로 나타낸 것
- Key를 지정함으로써 원하는 데이터를 빠르고 쉽게 찾아낼 수 있다
- 행과 열의 형태로 나타내며 각각 레코드(record), 속성(attribute)이라고도 부른다.

## 1.4 스키마와 데이터 타입

### 스키마

- 테이블의 뼈대를 구성하는 것
- 테이블에 대한 정보가 들어있다
- 테이블 정의 시 컬럼별로 저장할 수 있는 데이터 타입까지 명시해야 한다.

### 데이터 타입

- 데이터베이스 테이블에는 특정 데이터 유형이 있는 여러 열이 포함
- 프로그래밍 관점에서 데이터를 식별하기 위한 분류
- 문자열, 숫자, 날짜 등의 속성이 있다.
- 데이터 타입에 따라 사용하는 SQL 구문이 다르다.
