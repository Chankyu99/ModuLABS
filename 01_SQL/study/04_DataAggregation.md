# 4. 데이터 집계

## 학습 내용

- 집계함수(COUNT, SUM, AVG, MIN, MAX)
- 그룹화 (GROUP BY)
- 그룹화한 데이터 조건 주기 (HAVING)
    - WHERE과 HAVING의 차이점

## 4-1. 집계함수

- COUNT(*) : 행의 개수, NULL 값도 포함하기 때문에 컬럼 결측치 체크 불가
- COUNT(컬럼명) : NULL 값을 제외한 행의 개수 카운트

- SUM(컬럼명) : 컬럼 내 모든 값의 합계
- AVG(컬럼명) : 컬럼 값 평균
- MIN(컬럼명) : 컬럼 내 최솟값
- MAX(컬럼명) : 컬럼 내 최댓값

### 특징

- 집계함수 내부에 조건문을 사용 할수있다   
- 집계함수의 결과값끼리 연산이 가능하다

## 4-2. 데이터 그룹화 하기 (GROUP BY)

### GROUP BY

- 집계함수와 쓰임
- 특정 컬럼을 기준으로 그룹화
- SELECT 절에 컬럼명, 집계함수가 같이 쓰인경우, 컬럼명은 GROUP BY에 쓰여야한다
- GROUP BY에 지정한 컬럼 이외의 컬럼은 집계함수를 사용하지 않은 채 SELECT 구문에 지정할 수 없다 (어떤 행을 출력해야하는지 모르기 떄문)
- 결과값은 GROUP BY절에 쓰인 컬럼 항목들의 행 개수에 의해 결정

### HAVING 

- 집계함수를 사용해 조건을 비교할 때 사용 (그룹의 결과를 제한하는 역할)
- GROUP BY로 그룹화된 결과에 조건을 적용할 때 사용
- 집계함수, 데이터 그룹핑을 통해 데이터를 특성별로 분류하거나 계산 할 수 있다

### WHERE과 HAVING의 차이점

- WHERE은 행을 필터링, HAVING은 그룹을 필터링
- WHERE은 집계함수를 사용할 수 없음!, GROUP BY 전에 사용된다

> 예시 : '부서별 평균 급여' -> '퇴사자를 제외'하고 싶다면 WHERE를 써서 행에서 1차 필터링 -> '평균 급여가 500만 원 이상인 부서'만 보고 싶을때 HAVING사용
