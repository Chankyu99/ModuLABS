## 5-1. 기본키(PK), 외래키(FK)

### 기본키 (Primary Key)

- 테이블이 가진 고유의 기본키 -> 이것을 통해 DB 내 다수의 테이블이 서로 연결되어있다
- 중복된 값, NULL 값을 가질 수 없음
- 테이블당 PK로 하나의 컬럼이 지정되어 있음

> 이렇게 기본키를 주는 이유 : 테이블이 하나로 합쳐져 있다면, 값을 수정할 때 모든 테이블의 데이터를 전부 바꿔야하는 문제가 발생한다. 하지만 기본키를 통해 테이블을 분리하면, 기본키만 수정하면 되기 때문에 데이터의 무결성을 지킬 수 있다.

### 외래키 (Foreign Key)

- JOIN으로 다른 테이블과 연결 시 다른 테이블의 기본키를 참조하는 컬럼
- 참조하는 테이블의 기본키와 같은 값을 가지거나 NULL 값을 가질 수 있음
- 외래키는 참조하는 테이블의 기본키와 동일한 값의 종류와 범위를 가진다
- 테이블당 여러 개가 존재할 수 있음

## 5-2. 다중 테이블 사용하기(JOIN)

### JOIN

- RDB에서 1개 이상의 테이블과 테이블을 연결하는 방법
- 기본적으로 JOIN 유형은 내부 조인(INNER JOIN)을 실행한다

### 2개 테이블 JOIN

```sql
SELECT 컬럼명       -- 출력될 컬럼
FROM 테이블명 AS A  -- 첫번째 테이블
JOIN 테이블명 AS B  -- 두번째 테이블
ON A.id = B.id      -- 연결할 조건
```

### JOIN 종류

- 어느 부분을 가져올 것인가에 따라 JOIN의 종류가 나뉜다
    - INNER JOIN : 두 테이블의 교집합, ON 절의 조건을 만족한 데이터만 가져와 출력 (일치하는게 없다면 가져오지 않는다)
    - OUTER JOIN : 합집합, 조건을 만족하지 않는 데이터도 가져온다 (일치하지 않는 데이터가 있다면 NULL로 표시)
        - LEFT JOIN : 왼쪽 테이블 기준, 왼쪽 테이블의 데이터를 모두 가져온다 (쿼리를 왼쪽에서 오른쪽으로 읽으니 자주 사용)
        - RIGHT JOIN : 오른쪽 테이블 기준, 오른쪽 테이블의 데이터를 모두 가져온다
        - FULL OUTER JOIN : 합집합 (데이터 누락 체크시 활용)
    - CROSS JOIN : 모든 경우의 수 (테스트 데이터를 대량으로 만들 때 활용)
    - SELF JOIN : 자기 자신과 조인 (테이블 내 계층 구조를 표현할 때 활용)

- JOIN 사용 시 별칭도 사용 가능 
- 가장 중심이 되는 테이블을 가장 먼저 작성

## 5-3. 데이터 이어붙이기(UNION)

- 테이블을 아래로, 세로로 붙이는 형태
- 데이터를 합치며 새로운 행을 생성
- 데이터가 너무 많으면 분할해서 저장한 테이블을 합칠 때 사용
- 컬럼의 개수와 데이터 타입이 같아야 한다
- UNION ALL은 중복을 허용, UNION은 중복을 제거

```sql
SELECT *
FROM 테이블1
UNION ALL -- ALL을 뺴면 중복을 제거한다
SELECT *
FROM 테이블2
```

## 5-4. 서브쿼리 원리와 방식(Subquery)

- 하나의 쿼리문 안에 또 다른 쿼리문을 작성하는 방식
- 여러번의 쿼리를 한 번의 쿼리로 작성 가능
- JOIN과 달리 새로운 컬럼이 필요하지 않을 때 사용 (완전히 다른 테이블에서 데이터 값을 조회하여 메인쿼리의 조건으로 사용하려할 때 유용)
- 서브쿼리 -> 메인쿼리 순으로 실행된다

```sql
SELECT *
FROM 테이블명
WHERE 컬럼명 IN (SELECT 컬럼명 FROM 테이블명) -- 괄호로 묶어서 사용, 연산자의 오른쪽, ORDER BY 사용 불가
```

### WHERE절 서브쿼리

- WHERE 절에서 서브쿼리의 결과를 사용하여 메인쿼리의 조건을 지정
- 단일행, 다중행 서브쿼리로 나뉜다
    - 단일행 서브쿼리 : 서브쿼리의 결과가 1개일 때 ( = , != , > , < , >= , <= 사용)
    - 다중행 서브쿼리 : 서브쿼리의 결과가 여러 개일 때 (IN, NOT IN, ANY, ALL 사용)

### FROM절 서브쿼리

- 뷰(view)처럼 작동하여 인라인뷰(Inline View)라고 부른다

> 뷰(View) : 복잡한 쿼리나 보안이 필요한 데이터를 마치 하나의 일반 테이블처럼 편하게 쓰기 위해 만든 가짜 테이블

- 데이터 추출 결과를 FROM 절에서 하나의 테이블처럼 사용하고 싶을 때 사용
- FROM 절에 서브쿼리를 사용하면 특정 조건식을 갖는 데이터 추출결과를 테이블처럼 사용 가능
- 기존 메인 단일쿼리로는 테이블에서 각 제품의 최대 금액까지 알 수 있었다면 서브쿼리를 이용하면 최대 금액의 제품을 구매한 구매자를 확인할 수 있다
